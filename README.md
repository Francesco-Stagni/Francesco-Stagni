## How I Think About Problems

My default mode is cost minimization. If a task is repetitive, the question
isn't "how do I do this better". It's "why am I doing this at all."

Screening 200 emails a day to find 5 serious leads is not just a workflow problem but a design failure. 
The right solution is a system that does the screening and surfaces only what matters. 
Pushing one button beats reading a hundred messages.

My economics background trained me to apply this logic broadly: rental
inquiries, finance tracking, research pipelines, everyday tasks that quietly
eat time. If there's a more efficient path, the only real question is why
you're not already on it.
